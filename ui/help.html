Динамические цены

Модуль отслеживает занятость постов и при высокой нагрузке отправляет в MQTT коэффициент списания (не меняет прайс-лист).

Логика CRM

  • Опрашивает post-states и считает занятые посты выбранной автомойки.
  • При достижении порога — публикует {prefix}/{serial}/set/surge с коэффициентом (через message-processor, учётная запись system).
  • При снижении занятости — отправляет сброс (active: 0, coefficient: 1).
  • Цены в CRM и на панели не перезаписываются.

MQTT payload

{ "coefficient": 1.10, "active": 1, "until_balance_zero": 1 }

Коэффициент действует на устройстве до обнуления баланса текущей сессии (логика на стороне панели).

Внедрение на автомойке

Подробная спецификация для прошивки панели — файл CLIENT.md в репозитории модуля.

Настройки

  • wash_id — автомойка.
  • busy_threshold — минимум занятых постов (например, 9 из 10).
  • price_increase_percent — процент → коэффициент (10% = 1.10).
  • poll_interval — интервал опроса (от 5 сек, шаг 5 сек).
